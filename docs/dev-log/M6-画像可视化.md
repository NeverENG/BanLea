# M6 · 画像可视化（阶段记录）
> 日期：2026-06-10 · 对应计划 §3.2 / §4 / §13 M6
> 状态：已开始。M5 已完成推荐信息流最小闭环，画像可视化可以复用 portrait timeline、latest portrait 和 evidence timeline。

## 本阶段目标（计划 M6）
雷达图 + 版本演化 + AI 协商修改，形成入口二的“我的画像”视图基础。

## 起点
- 已有 `Portrait` schema：
  - 27 维画像；
  - 每维独立 `summary / confidence / evidenceIds`；
  - 数值维可携带 `score`，标签维可携带 `tags`。
- 已有 `DIMENSION_META`：
  - 维度中文标签；
  - 主/子 harness 层；
  - 变化速度；
  - 功能分组。
- 已有画像状态与版本列表：
  - `PortraitStatusPanel`；
  - `loadPortraitTimeline`；
  - `DomainLearningSnapshot.status.latest`。

## 优先增量
1. 维度可视化 view model：把 latest portrait 转为稳定的维度条目，包含 label、score、confidence、summary 和低可信度标记。
2. 画像概览 UI：在现有画像状态入口展示维度概览，先用条形图/卡片承接，后续升级雷达图。
3. 版本演化增强：展示关键维度随版本变化。
4. AI 协商修改：先建提案模型与 evidence 写回，再接 UI。

## 已完成增量一：维度概览可视化基础
- 扩展 `features/portrait`：
  - 新增 `buildPortraitDimensionVisualItems`；
  - 将 latest portrait dimensions 映射为带 label、score、confidence、summary、evidenceCount 的 view model；
  - 接入 `DIMENSION_META`，保留 harness、tier 和 groups 元数据；
  - 支持低可信度维度标记，用于 UI 弱化。
- 扩展 `PortraitStatusPanel`：
  - 在画像状态下新增“维度概览”；
  - 使用条形进度展示维度值；
  - 展示维度摘要、值和 confidence；
  - 低可信度维度降低视觉权重。
- 新增单测：
  - 验证维度 view model 元数据映射；
  - 验证低可信度标记；
  - 验证未知维度过滤。

## 已完成增量二：关键维度版本趋势
- 扩展 `PortraitTimelineItem`：
  - 携带每个版本的 `dimensions` 快照；
  - 保留原有版本列表展示字段。
- 扩展 `features/portrait`：
  - 新增 `buildPortraitDimensionTrendItems`；
  - 默认提取 `mastery / progress / interest / velocity` 关键维度；
  - 输出按版本升序排列的趋势点、latestValue 与 delta。
- 扩展 `PortraitStatusPanel`：
  - 新增“版本趋势”区域；
  - 使用紧凑柱状条展示关键维度随版本变化；
  - 展示每个维度的变化量。
- 新增单测：
  - 验证趋势点按版本升序输出；
  - 验证 latestValue 与 delta 计算；
  - 验证缺失维度不会生成趋势项。

## 已完成增量三：画像协商修改提案模型
- 扩展 `features/portrait`：
  - 新增 `buildPortraitRevisionEvidenceDraft`；
  - 将用户对单个画像维度的修改请求转换为 self-report evidence 草稿；
  - 草稿携带目标维度 `dimensionHints`；
  - confidenceScore 带 clamp，避免越界。
- 新增 `recordPortraitRevisionRequest`：
  - 复用现有 `LearningEventService.recordSelfReport`；
  - 协商修改不会直接覆盖画像，而是写入 evidence 后走画像重评估闭环。
- 新增单测：
  - 验证协商请求的 statement、summary、dimensionHints 和 confidence；
  - 验证协商请求会通过 self-report evidence 写回。

## 已完成增量四：画像协商修改 UI 入口
- 扩展 `PortraitStatusPanel`：
  - 新增“协商修改”区域；
  - 从维度概览中选择目标维度；
  - 输入修改请求并提交；
  - 提交时展示忙碌与结果状态。
- 更新首页：
  - 新增画像协商提交状态；
  - 调用 `recordPortraitRevisionRequest` 写入 self-report evidence；
  - 提交后刷新 domain 快照与画像闭环状态。
