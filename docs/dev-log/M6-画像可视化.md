# M6 · 画像可视化（阶段记录）
> 日期：2026-06-10 · 对应计划 §3.2 / §4 / §13 M6
> 状态：已开始。M5 已完成推荐信息流最小闭环，画像可视化可以复用 portrait timeline、latest portrait 和 evidence timeline。

## 本阶段目标（计划 M6）
雷达图 + 版本演化 + AI 协商修改，形成入口二的“我的画像”视图基础。

## 起点
- 已有 `Portrait` schema：
  - 27 维画像；
  - 每维独立 `summary / confidence / evidenceIds`；
  - 数值维可携带 `score`，标签维可携带 `tags`。
- 已有 `DIMENSION_META`：
  - 维度中文标签；
  - 主/子 harness 层；
  - 变化速度；
  - 功能分组。
- 已有画像状态与版本列表：
  - `PortraitStatusPanel`；
  - `loadPortraitTimeline`；
  - `DomainLearningSnapshot.status.latest`。

## 优先增量
1. 维度可视化 view model：把 latest portrait 转为稳定的维度条目，包含 label、score、confidence、summary 和低可信度标记。
2. 画像概览 UI：在现有画像状态入口展示维度概览，先用条形图/卡片承接，后续升级雷达图。
3. 版本演化增强：展示关键维度随版本变化。
4. AI 协商修改：先建提案模型与 evidence 写回，再接 UI。

## 已完成增量一：维度概览可视化基础
- 扩展 `features/portrait`：
  - 新增 `buildPortraitDimensionVisualItems`；
  - 将 latest portrait dimensions 映射为带 label、score、confidence、summary、evidenceCount 的 view model；
  - 接入 `DIMENSION_META`，保留 harness、tier 和 groups 元数据；
  - 支持低可信度维度标记，用于 UI 弱化。

  - 低可信度维度降低视觉权重。
- 新增单测：
  - 验证维度 view model 元数据映射；
  - 验证低可信度标记；
  - 验证未知维度过滤。
