# M7 · 资料源适配层（阶段记录）
> 日期：2026-06-10 · 对应计划 §7 / §13 M7
> 状态：已开始。M6 已完成画像可视化最小闭环，资料源可复用画像上下文、推荐候选和书单入口。

## 本阶段目标（计划 M7）
建立 `ResourceSource` 可插拔适配层，优先接 GitHub 与合规 web/docs 资料源，把外部资料推荐打通到书单。

## 起点
- 已有书单仓储与 UI：
  - `reading_list` 可保存资料；
  - tutor 资源建议可写入书单；
  - 阅读状态可写回 evidence。
- 已有推荐信息流：
  - `read` 候选可从书单生成；
  - 后续资料源可为 `read` 候选补充真实 URL 与来源。
- 计划约束：
  - 不内置爬虫；
  - GitHub 使用官方 REST API；
  - 知乎仅预留官方 OAuth，不做非合规抓取。

## 优先增量
1. `ResourceSource` 核心接口与聚合器。
2. GitHub 资料源 query builder 与 REST client。
3. 将资料源结果转为 tutor resource suggestions / reading list items。
4. 资料源开关与状态展示。

## 已完成增量一：资料源核心接口
- 实现 `core/sources`：
  - 新增 `ResourceSource`；
  - 新增 `ResourceItem` 与搜索上下文；
  - 新增 `searchResourceSources`，过滤 disabled source 并按 source 限制结果数；
  - 新增 `flattenResourceSourceResults`；
  - 新增 `createStaticResourceSource`，便于本地测试与后续 mock。
- 新增单测：
  - 验证静态 source 按 topic 过滤；
  - 验证只查询 enabled source；
  - 验证 per-source limit 与 flatten limit。

## 已完成增量二：GitHub 资料源适配器
- 新增 `core/sources/github`：
  - 新增 GitHub repository search query builder；
  - 使用官方 REST API `/search/repositories`，默认按 stars 降序；
  - 支持可注入 `fetchJson`，测试不触发真实网络；
  - 支持可选 token，自动带 `Accept`、`Authorization` 与 GitHub API version headers；
  - 将 repository 结果映射为 `ResourceItem`，包含 stars、language、topics、updatedAt 等 metadata。
- 新增单测：
  - 验证 query 包含学习主题、practice intent 与安全限定符；
  - 验证 REST URL、排序参数、limit 与官方 headers；
  - 验证 GitHub repository 到通用资料项的转换。

## 已完成增量三：资料源结果转书单入口
- 扩展 tutor 资源建议：
  - `TutorResourceSuggestion` 支持可选 `sourceId`；
  - source-backed suggestion 写入书单时保留原始资料源 ID，不再丢失为临时 tutor ID。
- 新增 reading-list 转换 helper：
  - `resourceItemToTutorResourceSuggestion` / `resourceItemsToTutorResourceSuggestions`；
  - `resourceItemToReadingListItem` / `resourceItemsToReadingListItems`；
  - `addResourceItemsToReadingList`，可直接把资料源结果写入现有书单仓储；
  - `paper` 类型映射到书单 `article`，其余 article/video/repo/doc 保持原类型。
- 新增单测：
  - 验证资料源结果转换为 tutor suggestions 并保留 sourceId；
  - 验证资料源结果转换为 reading list insert rows；
  - 验证直接写入书单与 limit；
  - 验证 source-backed tutor suggestions 写入书单时保留原始 sourceId。

## 已完成增量四：资料源开关与状态展示
- 新增 `features/settings/resourceSourceSettings`：
  - 定义 GitHub、通用网页、文档站、arXiv、视频、知乎资料源状态；
  - GitHub 默认为 ready + enabled，其余合规源先作为 planned 预留；
  - 支持 localStorage 保存启用开关，不保存第三方凭据；
  - 新增 `enabledResourceSourceIds`，只返回已启用且 ready 的资料源。
- 新增 `ResourceSourceSettingsPanel`：
  - 在右侧设置栏展示每个资料源的启用状态、可用性与凭据模式；
  - 支持刷新与启用/禁用切换；
  - 接入 `App` 初始化加载与状态消息。
- 新增单测：
  - 验证默认 GitHub 可用；
  - 验证开关持久化；
  - 验证 planned source 即使启用也不会进入 usable 列表。
