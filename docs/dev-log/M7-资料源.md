# M7 · 资料源适配层（阶段记录）
> 日期：2026-06-10 · 对应计划 §7 / §13 M7
> 状态：已开始。M6 已完成画像可视化最小闭环，资料源可复用画像上下文、推荐候选和书单入口。

## 本阶段目标（计划 M7）
建立 `ResourceSource` 可插拔适配层，优先接 GitHub 与合规 web/docs 资料源，把外部资料推荐打通到书单。

## 起点
- 已有书单仓储与 UI：
  - `reading_list` 可保存资料；
  - tutor 资源建议可写入书单；
  - 阅读状态可写回 evidence。
- 已有推荐信息流：
  - `read` 候选可从书单生成；
  - 后续资料源可为 `read` 候选补充真实 URL 与来源。
- 计划约束：
  - 不内置爬虫；
  - GitHub 使用官方 REST API；
  - 知乎仅预留官方 OAuth，不做非合规抓取。

## 优先增量
1. `ResourceSource` 核心接口与聚合器。
2. GitHub 资料源 query builder 与 REST client。
3. 将资料源结果转为 tutor resource suggestions / reading list items。
4. 资料源开关与状态展示。

## 已完成增量一：资料源核心接口
- 实现 `core/sources`：
  - 新增 `ResourceSource`；
  - 新增 `ResourceItem` 与搜索上下文；
  - 新增 `searchResourceSources`，过滤 disabled source 并按 source 限制结果数；
  - 新增 `flattenResourceSourceResults`；
  - 新增 `createStaticResourceSource`，便于本地测试与后续 mock。
- 新增单测：
  - 验证静态 source 按 topic 过滤；
  - 验证只查询 enabled source；
  - 验证 per-source limit 与 flatten limit。

## 已完成增量二：GitHub 资料源适配器
- 新增 `core/sources/github`：
  - 新增 GitHub repository search query builder；
  - 使用官方 REST API `/search/repositories`，默认按 stars 降序；
  - 支持可注入 `fetchJson`，测试不触发真实网络；
  - 支持可选 token，自动带 `Accept`、`Authorization` 与 GitHub API version headers；
  - 将 repository 结果映射为 `ResourceItem`，包含 stars、language、topics、updatedAt 等 metadata。
- 新增单测：
  - 验证 query 包含学习主题、practice intent 与安全限定符；
  - 验证 REST URL、排序参数、limit 与官方 headers；
  - 验证 GitHub repository 到通用资料项的转换。
