## 本阶段目标（计划 M4）
待读书单（分类 / 已读 / 停留）+ 数据看板，让入口二的“书单与学习状态”可独立查看，并为后续推荐流提供稳定数据基础。

## 起点
- 已有 `reading_list` 表与 `ReadingListRepository`：
  - 插入 tutor 资源建议；
  - 按 domain 查询；
  - 更新状态并写回 reading evidence。
- 已有 evidence / portrait timeline：
  - 可展示当前 domain 最近证据；
  - 可展示画像版本演化。
- 当前首页右栏已有嵌入式书单，但还不是独立 M4 视图。

## 优先增量
1. 书单统计摘要：按 domain 汇总 todo / later / done / reading 数量和已读停留时长。
2. 书单视图模型：输出可直接给 UI 渲染的分组列表。
3. 看板摘要：基于 evidence、reading_list、portrait timeline 生成学习状态指标。
4. 首页右栏降重：保留主入口必要信息，把完整书单/看板能力下沉到 feature 层。

## 已完成增量一：书单统计摘要
- 扩展 `features/reading-list`：
  - 新增 `ReadingListSummary`；
  - 新增 `summarizeReadingList`，按 `todo / reading / done / later` 汇总数量；
  - 统计 `doneDwellSeconds`，只累计已读项停留时长；
  - 新增 `loadReadingListOverview`，一次读取列表与摘要。
- 更新首页右栏：
  - 待读书单区域显示全部、待读、稍后、已读和已读停留秒数；
  - 原列表和状态操作保留。
- 新增单测：
  - 验证状态计数；
  - 验证已读停留时长只累计 done 项；
  - 验证 overview 一次返回列表和摘要。

## 已完成增量二：书单分组视图模型
- 扩展 `features/reading-list`：
  - 新增 `ReadingListGroup`；
