# M8 · BanLea 界面重设计 PRD
> 日期：2026-06-10 · 对应计划 M8 打磨发布
> 状态：进行中。目标是把当前工程调试式三栏 UI 改为可交付的学习工作台。

## 问题判断
当前界面信息堆叠严重，左侧领域、中央功能、右侧设置/画像/书单全部同时出现，导致：
- 首屏没有明确主任务；
- 右侧栏承担太多功能，像调试面板；
- 数据看板、画像、关于我/API Key 缺少一级入口；
- 宽屏没有充分利用空间，窄屏也缺少自然折叠；
- 视觉语言偏默认 Tailwind 原型，没有产品气质。

## 设计目标
1. 一屏自适应全满：页面主体占满 viewport，用一个完整工作台承载所有功能。
2. 顶部一级导航：学习台、数据看板、我的画像、关于我。
3. 学习台二级切换：辅导、书单、推荐，避免把所有学习流混在一起。
4. 关于我承载个人设置：领域、自评建档、API Key、资料源开关。
5. 我的画像成为独立页面：画像状态、雷达图、协商修改和证据流不再挤在右栏。
6. 视觉克制但有质感：中性色底、绿色主行动、蓝色信息、暖色提示，避免单一紫色原型感。

## 信息架构
```text
BanLea App Shell
├─ 顶部栏
│  ├─ 品牌 / 当前领域 / 状态摘要
│  ├─ 领域切换
│  └─ 一级导航：学习台 / 数据看板 / 我的画像 / 关于我
├─ 学习台
│  ├─ 二级导航：辅导 / 书单 / 推荐
│  ├─ 主工作区：当前学习任务
│  └─ 侧栏：快速记录 / 本轮验证 / 摘要
├─ 数据看板
│  └─ 全宽数据与时间线
├─ 我的画像
│  ├─ 画像状态与维度
│  └─ 最近证据
└─ 关于我
   ├─ API Key
   ├─ 学习偏好
   └─ 资料源开关
```

## 首轮验收标准
- 桌面端首屏为单一全屏工作台，不再出现左右两块常驻杂乱侧栏。
- 顶部可直接切换到数据看板、我的画像、关于我。
- 关于我页能填写 API Key，并能管理冷启动和资料源。
- 学习台内仍能发送 tutor 消息、查看书单、查看推荐。
- `npm run typecheck`、`npm test`、`npm run build` 通过。
- 浏览器检查桌面与窄屏不出现明显重叠或空白主场景。

## 学习文件夹修正（2026-06-11）
- 产品语义：原“领域”改为用户自建的“学习文件夹”；每个学习文件夹对应一个子 harness 上下文。
- 数据落点：复用现有 `domains` 表和各业务表的 `domain_id`，不改动证据、画像、书单、推荐的底层关联方式。
- 已新增 `DomainRepository`：
  - `ensureDefaults` 初始化默认学习文件夹；
  - `list` 读取文件夹；
  - `insert` 新建用户文件夹。
- App 启动时会加载学习文件夹；新建后自动切换到该文件夹，后续对话、书单、画像、推荐都会使用新的 `domain_id`。
- 浏览器预览没有 Tauri SQL 运行时，因此增加 localStorage fallback；桌面端仍以 SQLite `domains` 表为准。

## 用户面收束（2026-06-11）
- 一级入口收束为「学习 / 资料 / 我的」：
  - 「学习」专注对话输入与历史消息；
  - 「资料」保留书单与推荐链接；
  - 「我的」承载 API Key、学习文件夹和学习偏好。
- 「数据看板 / 我的画像」从一级导航降级到「我的」里的进阶视图，保留能力但不作为普通用户的默认路径。
- 主对话区不再展示「检验一下 / 未掌握 / 已掌握」测验卡；掌握判断继续作为后续 AI 提示词和 evidence 机制的一部分演进。
- 推荐流隐藏内部排序细节：
  - 不再展示候选种子数、特征明细和本地打分；
  - 用户只看到推荐主题、推荐理由和「看过了 / 不感兴趣」反馈。
- 书单隐藏已读停留秒数，保留待读、阅读中、稍后、已读等真实用户操作状态。
- 「冷启动建档」文案改为「学习偏好」，避免把内部实现名暴露给用户。

## 手动资料链接（2026-06-12）
- 「资料 / 书单」页新增手动添加资料入口：
  - 用户可粘贴资料链接，标题可选；
  - 未带协议的链接会按 `https://` 处理；
  - 未填写标题时从 URL 域名和路径推断标题；
  - 未显式指定类型时按链接自动推断：GitHub 为代码，YouTube/Bilibili 为视频，文章路径为文章，其余为资料；
  - 新资料默认进入当前学习文件夹的待读清单。
- 桌面端写入 SQLite `reading_list` 表，`source_id` 使用 `manual:<addedAt>`，继续沿用现有 `domain_id` 隔离。
- 浏览器预览没有 Tauri SQL 时，会临时追加到当前页面，便于开发期直接验收。
- 当前增量只保存 URL 和标题，不新增备注字段，避免为低确定性信息改动迁移结构。
- 资料写入会先读取当前学习文件夹已有资料：
  - 手动添加时，规范化后的 URL 或 sourceId 已存在会直接返回旧条目；
  - tutor 资料建议和资料源直写时，同 URL 或同 sourceId 的条目会跳过；
  - 同一批资料源结果内部也会按 URL/sourceId 去重，避免上下文和书单被重复链接污染。

## 资料上下文注入对话（2026-06-12）
- Tutor prompt context 增加 `reading_resources` 段，注入当前学习文件夹的未完成资料。
- 注入范围：
  - 只读取当前 `domain_id`；
  - 排除 `done` 资料，避免已读项长期干扰；
  - 默认最多注入 6 条，控制 prompt 噪声。
- 当前只注入标题、URL、类型和状态，不抓取网页正文；后续资料解析可以独立做，不阻塞主对话链路。
- 没有 API Key 时，本地兜底回复也会引用第一条未完成资料，避免用户添加资料后在对话主链路里没有反馈。

## DeepSeek 对话适配（2026-06-12）
- API Key 设置增加 Provider 切换：Claude / DeepSeek。
- Keychain 按 provider 分开存储：
  - Claude 使用 `anthropic_api_key`；
  - DeepSeek 使用 `deepseek_api_key`。
- LLM client 增加 DeepSeek OpenAI-compatible `/chat/completions` 适配：
  - `light` → `deepseek-v4-flash`；
  - `deep` → `deepseek-v4-pro`。
- API Key settings service 调整为保存后立即初始化当前 provider，调用方不再需要额外读取 keychain 才能使用刚保存的 key。
- 当前 DeepSeek 只用于提问式对话回复；画像 live update / structured output 仍限定 Claude，避免把非等价结构化能力混入 harness 更新链路。
