# M2 · 自迭代闭环（阶段记录）

> 日期：2026-06-09 ｜ 对应计划 §13 M2
> 状态：**已开始**。M1 已提供画像 schema、harness 内核、仓储、触发器与触发式更新入口；M2 开始把真实用户行为接入证据流。

## 本阶段目标（计划 M2）
证据采集 + 触发器 + 重评估 + 版本演化 + 可解释变化，跑通 §5 自迭代闭环。

## 已完成增量一：证据采集服务
- 实现 `createEvidenceCollector`：
  - `recordChat`：记录用户/助手对话证据；
  - `recordSelfReport`：记录用户自评证据；
  - `recordQuiz`：记录测验结果证据；
  - `recordReading`：记录阅读/书单行为证据；
  - `recordRecommendationClick`：记录推荐点击与停留证据；
  - `recordRecommendationSkip`：记录推荐跳过证据。
- 采集服务统一生成 `NewEvidence` 并写入 `EvidenceRepository`：
  - 自动填充 `createdAt`；
  - 保留人类可读 `summary`；
  - 将结构化细节写入 `payload`；
  - 支持调用方覆盖 summary。
- 新增采集服务单测，覆盖六类证据事件。

## 已完成增量二：采集后按需更新入口
- 实现 `recordEvidenceAndMaybeUpdate`：
  - 先将一条标准 `NewEvidence` 写入 `EvidenceRepository`；
  - 随后调用 `runHarnessUpdateIfTriggered`；
  - 若触发条件未满足，则不调用 LLM、不写画像版本、不消费证据；
  - 若触发条件满足，则完成画像生成/局部重评估、版本落库和证据消费；
  - 返回已写入证据与画像更新结果，便于 UI 展示或后台任务记录。
- 新增端到端入口单测：
  - 写入证据后触发条件未满足；
  - 写入证据后首次建档触发并消费证据。

## 已完成增量三：学习事件服务层
- 新增 `features/events`：
  - 面向 UI/功能切片暴露 `recordChat`、`recordSelfReport`、`recordQuiz`、`recordReading`、`recordRecommendationClick`、`recordRecommendationSkip`；
  - 内部先调用 evidence 采集器写入证据；
  - 再调用 `runHarnessUpdateIfTriggered` 按需更新画像；
  - 根据 `domain === "global"` 自动选择主画像，其余 domain 自动选择子画像；
  - 返回已写入证据和画像更新结果，便于界面显示“已记录/已触发画像更新/继续积累证据”。
- 新增学习事件服务单测：
  - 聊天事件未达到触发条件时只记录证据；
  - 推荐点击长停留触发画像重评估；
  - global 自评事件走主画像更新。

## 已完成增量四：开发期事件采集面板
- 将首页从脚手架占位替换为 M2 事件采集面板：
  - 左侧选择 domain；
  - 中央记录 chat / self_report / quiz / reading / reco_click / reco_skip；
  - 右侧展示写入状态和最近 evidence；
  - 点击记录时使用真实 `EvidenceRepository` 写入本地 SQLite。
- 设计边界：
  - 当前面板只负责 evidence 写入，避免无 API Key 时误触发 Claude；
  - 画像更新仍通过已实现的服务层入口接入后续正式交互。

## 已完成增量五：无 API Key 安全闭环模式
- 扩展 `runHarnessUpdateIfTriggered`：
  - 触发条件满足但模型不可用时返回 `deferred`；
  - 不调用 LLM、不写画像版本、不消费证据；
  - 有注入模型或 Claude client 已初始化时才执行画像更新。
- 扩展 `features/events`：
  - 支持 `updateAfterEvidence` 注入，避免 UI 静态依赖 LLM/harness 实现；
  - 事件服务返回 skipped / deferred / updated 三类结果。
- 更新开发期事件采集面板：
  - 面板改为调用 `createLearningEventService`；
  - 同时接入 `evidenceRepo` 与 `portraitRepo`；
  - 无 API Key 时仍可写入 evidence，并展示“等待 API Key 初始化后更新画像”。
- 新增安全模式单测：
  - harness 层触发但模型不可用时 deferred；
  - event service 层模型不可用时只记录 evidence。

## 已完成增量六：前端 API Key 设置
- 在开发期面板右侧增加 API Key 区域：
  - 读取 keychain 状态；
  - 保存 Anthropic API Key；
  - 删除已保存 Key；
  - 刷新当前状态；
  - 只显示 masked key，不在界面展示完整密钥。
- 新增 `features/settings/apiKeySettings`：
  - 封装 `getApiKey` / `saveApiKey` / `deleteApiKey`；
  - 支持 store 注入，便于无 Tauri 环境单测；
  - 保存前 trim，拒绝空 key。
- 说明：本增量只完成前端 Key 管理，不初始化 Claude client；避免重新把 Anthropic SDK 静态打入前端包。live 调用验证仍待后续接入。
