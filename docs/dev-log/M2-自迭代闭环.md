# M2 · 自迭代闭环（阶段记录）

> 日期：2026-06-09 ｜ 对应计划 §13 M2
> 状态：**已开始**。M1 已提供画像 schema、harness 内核、仓储、触发器与触发式更新入口；M2 开始把真实用户行为接入证据流。

## 本阶段目标（计划 M2）
证据采集 + 触发器 + 重评估 + 版本演化 + 可解释变化，跑通 §5 自迭代闭环。

## 已完成增量一：证据采集服务
- 实现 `createEvidenceCollector`：
  - `recordChat`：记录用户/助手对话证据；
  - `recordSelfReport`：记录用户自评证据；
  - `recordQuiz`：记录测验结果证据；
  - `recordReading`：记录阅读/书单行为证据；
  - `recordRecommendationClick`：记录推荐点击与停留证据；
  - `recordRecommendationSkip`：记录推荐跳过证据。
- 采集服务统一生成 `NewEvidence` 并写入 `EvidenceRepository`：
  - 自动填充 `createdAt`；
  - 保留人类可读 `summary`；
  - 将结构化细节写入 `payload`；
  - 支持调用方覆盖 summary。
- 新增采集服务单测，覆盖六类证据事件。

## 已完成增量二：采集后按需更新入口
- 实现 `recordEvidenceAndMaybeUpdate`：
  - 先将一条标准 `NewEvidence` 写入 `EvidenceRepository`；
  - 随后调用 `runHarnessUpdateIfTriggered`；
  - 若触发条件未满足，则不调用 LLM、不写画像版本、不消费证据；
  - 若触发条件满足，则完成画像生成/局部重评估、版本落库和证据消费；
  - 返回已写入证据与画像更新结果，便于 UI 展示或后台任务记录。
- 新增端到端入口单测：
  - 写入证据后触发条件未满足；
  - 写入证据后首次建档触发并消费证据。
