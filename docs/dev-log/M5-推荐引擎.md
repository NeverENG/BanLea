# M5 · 推荐引擎（阶段记录）
> 日期：2026-06-09 · 对应计划 §6 / §13 M5
> 状态：已开始。M4 已提供 domain 快照、书单与看板独立入口，推荐流可以复用现有画像、证据和书单数据。

## 本阶段目标（计划 M5）
onboarding 种子建档（冷启动）+ 猜你想学/想看候选生成 + 本地排序 + 点击/停留双更新（画像 + 算法），形成入口二的推荐信息流基础。

## 起点
- 已有 `types/recommendation`：
  - `learn / read` 两类候选；
  - `interest_match / adjacency / mentioned / difficulty_fit / novelty` 五类排序特征；
  - 候选包含 `score / shownAt / clicked / dwellSeconds / skipped` 反馈字段。
- 已有推荐行为 evidence：
  - `recordRecommendationClick`；
  - `recordRecommendationSkip`；
  - 强推荐反馈会触发画像重评估。
- 已有 `reading_list`：
  - tutor 资源建议可入书单；
  - 书单状态可写回 reading evidence。
- `core/recommender` 目前为空入口，需要先补纯逻辑排序与反馈骨架。

## 优先增量
1. 本地排序器：输入候选、特征权重和探索参数，输出稳定排序与分数。
2. 反馈学习：点击、停留、跳过更新本地权重，避免只更新画像不更新排序。
3. 冷启动种子：onboarding 回答转为画像种子和首批方向。
4. 信息流视图：逐条展示猜你想学/想看，点击/跳过/停留写回 evidence。

## 已完成增量一：本地排序器骨架
- 实现 `core/recommender`：
  - 新增默认特征权重 `DEFAULT_RECOMMENDER_WEIGHTS`；
  - 新增 `scoreRecommendation`，按推荐特征计算候选分数；
  - 新增 `rankRecommendations`，输出稳定排序和 `rank`；
  - 支持 `noveltyBoost`，为探索项提供显式加权。
- 新增单测：
  - 验证默认权重分数；
  - 验证排序和 rank 写入；
  - 验证同分稳定排序；
  - 验证探索加权可以提升新鲜候选。

## 已完成增量二：反馈学习骨架
- 扩展 `core/recommender`：
  - 新增 `updateRecommendationWeights`；
  - 点击会按候选活跃特征提高权重；
  - 长停留点击比普通点击提供更强正反馈；
  - 跳过会按候选活跃特征降低权重；
  - 权重更新带上下界，避免反馈无限放大。
- 新增单测：
  - 验证点击增权；
  - 验证长停留更强；
  - 验证跳过降权；
  - 验证权重 clamp。

## 已完成增量三：候选输入建模
- 扩展 `core/recommender`：
  - 新增主题种子 `RecommendationTopicSeed`；
  - 新增书单种子 `RecommendationReadingSeed`；
  - 新增 `buildRecommendationCandidates`；
  - 支持从画像下一步、画像缺口、近期提问和手工种子生成 `learn` 候选；
  - 支持从未完成书单项生成 `read` 候选；
  - 过滤空种子和已读书单项，按 `kind + topic` 去重并合并特征；
  - 输出复用本地 ranker 的分数排序和 limit 截断。
- 新增单测：
  - 验证主题种子生成学习候选；
  - 验证未完成书单生成阅读候选；
  - 验证空项过滤、已读过滤和去重合并；
  - 验证输出按分数排序且 obey limit。

## 已完成增量四：feed 视图模型
- 新增 `features/feed`：
  - 新增 `buildFeedRecommendationView`；
  - 从 domain 快照提取画像 `nextFocus` 和 `gaps / misconceptions` 作为学习候选种子；
  - 从最近用户提问提取学习候选种子；
  - 从未完成书单项提取阅读候选种子；
  - 复用 `buildRecommendationCandidates` 输出可展示的信息流 item；
  - 提供来源计数与空状态原因。
- 新增单测：
  - 验证画像、近期提问、书单可生成 feed items；
  - 验证忽略 assistant 消息和已读书单项；
  - 验证 recent message limit 与输出 limit。

## 已完成增量五：推荐信息流主视图入口
- 新增 `features/feed/FeedWorkspaceView`：
  - 展示推荐候选数、主题种子数和书单种子数；
  - 展示 `learn / read` 信息流条目、推荐理由、分数和主要特征；
  - 提供刷新入口与空状态。
- 更新首页：
  - 主工作区新增“推荐”标签；
  - 缓存最近一次 `DomainLearningSnapshot`；
  - 使用 `buildFeedRecommendationView` 从同一份 domain 快照派生推荐信息流。
