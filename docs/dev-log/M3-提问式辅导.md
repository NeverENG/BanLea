# M3 · 提问式辅导（阶段记录）
> 日期：2026-06-09 · 对应计划 §13 M3
> 状态：已开始。M2 已提供 evidence 采集、触发器、deferred/live 闭环入口、画像状态与时间线基础。

## 本阶段目标（计划 M3）
画像注入的个性化辅导 + 学习计划 + 配套资料入书单 + 验证出题，让“提问式学习”成为主入口。

## 已完成增量一：本地 assistant 占位回复骨架
- 扩展 `features/tutor`：
  - `TutorMessage.role` 支持 `user | assistant`；
  - `sendUserMessage` 返回 `userMessage` 与 `assistantMessage`；
  - 保留旧的 `message` 字段指向 `userMessage`，避免现有调用方断裂；
  - assistant 回复先基于闭环结果生成本地占位内容。
- 更新首页主入口：
  - 用户发送消息后同时显示用户消息与 assistant 回复；
  - user 消息右侧高亮；
  - assistant 消息左侧中性样式；
  - 后续可将本地占位回复替换为画像注入后的 Claude 辅导输出。
- 新增/更新单测：
  - 发送用户消息仍会记录 chat evidence；
  - 返回用户消息；
  - 返回本地 assistant 占位回复。

## 已完成增量二：画像上下文构建
- 扩展 `features/tutor`：
  - 新增 `loadTutorPromptContext`；
  - 读取 `global` 主画像；
  - 读取当前 domain 子画像；
  - 对画像维度按置信度过滤、排序和截断；
  - 输出结构化 snapshot 与可直接放入 tutor prompt 的 `systemContext`。
- 上下文内容包含：
  - 当前 domain；
  - 主画像版本、整体可信度、变更摘要、高置信维度；
  - 子画像版本、整体可信度、变更摘要、高置信维度；
  - nextFocus。
- 新增单测：
  - 验证主画像与子画像同时读取；
  - 验证低置信维度被过滤；
  - 验证维度按置信度排序；
  - 验证 `global` domain 不重复读取子画像。

## 已完成增量三：辅导回复生成入口
- 扩展 `features/tutor`：
  - 新增 `TutorPromptContextProvider`，允许发送用户消息后读取最新画像上下文；
  - 新增 `TutorReplyGenerator`，把用户输入、学习闭环结果和画像上下文交给回复生成器；
  - 默认生成器继续使用本地占位回复，保证无 API Key 模式和旧调用方行为不变。
- 更新首页主入口：
  - 发送 tutor 消息时加载当前 domain 的画像上下文；
  - 先只接入上下文 provider，不在该提交里直接调用 Claude，保持生成边界可测试。
- 新增单测：
  - 验证自定义 generator 能收到归一化后的用户输入、`LearningEventResult` 和 `systemContext`；
  - 验证 generator 输出会成为 assistant 消息内容。

## 已完成增量四：第一版 Claude 个性化辅导回复
- 新增 `features/tutor/claudeReply`：
  - 通过 `TutorReplyGenerator` 接入 `core/llm.ask`；
  - 默认使用轻量模型档位生成短回复；
  - 将 `systemContext` 注入 system prompt；
  - 模型不可用、空回复或异常时回落到本地占位回复，避免证据已写入后整轮失败。
- 更新首页主入口：
  - Claude client 已初始化时动态加载 Claude generator；
  - 无 API Key 时不触发模型调用，继续使用本地回复。
- 新增单测：
  - 验证模型不可用时不调用 `ask`；
  - 验证模型可用时携带画像上下文、使用轻量档位；
  - 验证模型异常时返回本地回复和错误摘要。

## 已完成增量五：三段式辅导回复结构
- 扩展 `features/tutor`：
  - 新增 `TutorReplySections` 和 `formatTutorReplySections`；
  - 本地兜底回复统一输出“学习计划 / 关键解释 / 练习/验证问题”三段；
  - 保留原有闭环状态说明，但放入“关键解释”段落。
- 更新 Claude prompt：
  - 要求模型严格使用同样三段标题；
  - 标题独占一行，便于后续前端结构化渲染或拆分。
- 更新首页消息样式：
  - assistant 消息支持换行显示，避免三段式内容压成一行。

## 已完成增量六：配套资料入书单基础
- 新增 `types/readingList`：
  - 定义书单条目的 kind、status、schema 和新建输入类型；
  - 修正 zod default 场景下的新建输入类型，允许调用方省略默认字段。
- 新增 `db/readingListRepo`：
  - 封装 `reading_list` 表的插入和按 domain 查询；
  - 接入 `db/index` 的仓库入口。
- 新增 `features/reading-list`：
  - 提供 `loadReadingList` 给右栏展示；
  - 提供 `addTutorResourceSuggestions`，把 tutor 资源建议写入本地书单。
- 扩展 tutor turn：
  - 新增 `TutorResourceSuggestion` 和 `TutorResourceSuggestionProvider`；
  - 本地 provider 先生成一条 doc 占位资料，后续可替换为真实资源检索结果。
- 更新首页主入口：
  - 发送 tutor 消息后把资源建议写入 `reading_list`；
  - 右侧展示当前 domain 的待读书单前 5 条。

## 已完成增量七：书单状态回流 evidence
- 扩展 `db/readingListRepo`：
  - 新增 `updateStatus`，支持更新书单项状态、readAt 和 dwellSeconds；
  - 更新后返回最新书单项，供 feature 层继续写 evidence。
- 扩展 `features/reading-list`：
  - 新增 `changeReadingListItemStatus`；
  - 状态变更后可调用 `recordReading` 写回 reading evidence；
  - `done` 状态自动写入 readAt，`later` 等状态保持 readAt 为空。
- 更新首页右栏：
  - 每条待读资料提供“已读 / 稍后”动作；
  - 操作后刷新书单、证据、闭环状态和画像时间线。

## 已完成增量八：本地验证问题与 quiz evidence
- 扩展 tutor turn：
  - 新增 `TutorCheckQuestion` 和 `TutorCheckQuestionProvider`；
  - 本地 provider 根据本轮问题生成一个验证问题；
  - `sendUserMessage` 返回 `checkQuestion`，供 UI 暂存本轮验证任务。
- 更新首页右栏：
  - 展示“本轮验证”问题；
  - 用户可选择“未掌握 / 已掌握”；
  - 自测结果调用 `recordQuiz` 写入 quiz evidence，并刷新闭环状态。
- 当前阶段先记录分数信号，不做答案文本判分；后续可接 Claude 评分或结构化测验。

## 已完成增量九：会话历史最小持久化
- 新增 `db/tutorSessionRepo`：
  - 封装 `sessions` 和 `messages` 表；
  - 支持创建最近会话、更新会话时间、写入消息、按 session 读取消息。
- 新增 `features/history`：
  - `loadLatestTutorHistory` 读取当前 domain 最近一条会话；
  - `saveTutorTurnMessages` 将一轮 user/assistant 消息落库，并返回 UI 可用消息列表。
- 更新首页主入口：
  - domain 快照加载时同步最近会话消息；
  - 发送消息后先保存到 `messages`，再用持久化结果刷新聊天区；
  - `tutorSessionId` 暂存当前会话，避免同一 domain 的连续消息分散。
