# M3 · 提问式辅导（阶段记录）
> 日期：2026-06-09 · 对应计划 §13 M3
> 状态：已开始。M2 已提供 evidence 采集、触发器、deferred/live 闭环入口、画像状态与时间线基础。

## 本阶段目标（计划 M3）
画像注入的个性化辅导 + 学习计划 + 配套资料入书单 + 验证出题，让“提问式学习”成为主入口。

## 已完成增量一：本地 assistant 占位回复骨架
- 扩展 `features/tutor`：
  - `TutorMessage.role` 支持 `user | assistant`；
  - `sendUserMessage` 返回 `userMessage` 与 `assistantMessage`；
  - 保留旧的 `message` 字段指向 `userMessage`，避免现有调用方断裂；
  - assistant 回复先基于闭环结果生成本地占位内容。
- 更新首页主入口：
  - 用户发送消息后同时显示用户消息与 assistant 回复；
  - user 消息右侧高亮；
  - assistant 消息左侧中性样式；
  - 后续可将本地占位回复替换为画像注入后的 Claude 辅导输出。
- 新增/更新单测：
  - 发送用户消息仍会记录 chat evidence；
  - 返回用户消息；
  - 返回本地 assistant 占位回复。

## 已完成增量二：画像上下文构建
- 扩展 `features/tutor`：
  - 新增 `loadTutorPromptContext`；
  - 读取 `global` 主画像；
  - 读取当前 domain 子画像；
  - 对画像维度按置信度过滤、排序和截断；
  - 输出结构化 snapshot 与可直接放入 tutor prompt 的 `systemContext`。
- 上下文内容包含：
  - 当前 domain；
  - 主画像版本、整体可信度、变更摘要、高置信维度；
  - 子画像版本、整体可信度、变更摘要、高置信维度；
  - nextFocus。
- 新增单测：
  - 验证主画像与子画像同时读取；
  - 验证低置信维度被过滤；
  - 验证维度按置信度排序；
  - 验证 `global` domain 不重复读取子画像。

## 已完成增量三：辅导回复生成入口
- 扩展 `features/tutor`：
  - 新增 `TutorPromptContextProvider`，允许发送用户消息后读取最新画像上下文；
  - 新增 `TutorReplyGenerator`，把用户输入、学习闭环结果和画像上下文交给回复生成器；
  - 默认生成器继续使用本地占位回复，保证无 API Key 模式和旧调用方行为不变。
- 更新首页主入口：
  - 发送 tutor 消息时加载当前 domain 的画像上下文；
  - 先只接入上下文 provider，不在该提交里直接调用 Claude，保持生成边界可测试。
- 新增单测：
  - 验证自定义 generator 能收到归一化后的用户输入、`LearningEventResult` 和 `systemContext`；
  - 验证 generator 输出会成为 assistant 消息内容。
